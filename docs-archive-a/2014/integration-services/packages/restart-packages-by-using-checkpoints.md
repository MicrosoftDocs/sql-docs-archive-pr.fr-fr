  [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] <span data-ttu-id="9ff81-103">peut redémarrer les packages ayant échoué à partir du point d'échec, au lieu de reprendre l'exécution du package tout entier.</span><span class="sxs-lookup"><span data-stu-id="9ff81-103">can restart failed packages from the point of failure, instead of rerunning the whole package.</span></span> <span data-ttu-id="9ff81-104">Si un package est configuré pour utiliser des points de contrôle, des informations sur l'exécution du package sont écrites dans un fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-104">If a package is configured to use checkpoints, information about package execution is written to a checkpoint file.</span></span> <span data-ttu-id="9ff81-105">Lorsque le package ayant échoué est relancé, le fichier de point de contrôle est utilisé pour redémarrer le package à partir du point d'échec.</span><span class="sxs-lookup"><span data-stu-id="9ff81-105">When the failed package is rerun, the checkpoint file is used to restart the package from the point of failure.</span></span> <span data-ttu-id="9ff81-106">Si le package est exécuté avec succès, le fichier de point de contrôle est supprimé, puis recréé à l’exécution suivante du package.</span><span class="sxs-lookup"><span data-stu-id="9ff81-106">If the package runs successfully, the checkpoint file is deleted, and then re-created the next time the package is run.</span></span>  
  
 <span data-ttu-id="9ff81-107">L'utilisation des points de contrôle dans un package offre les avantages suivants :</span><span class="sxs-lookup"><span data-stu-id="9ff81-107">Using checkpoints in a package can provide the following benefits.</span></span>  
  
-   <span data-ttu-id="9ff81-108">Éviter de répéter le téléchargement ascendant et descendant de gros fichiers.</span><span class="sxs-lookup"><span data-stu-id="9ff81-108">Avoid repeating the downloading and uploading of large files.</span></span> <span data-ttu-id="9ff81-109">Par exemple, un package qui télécharge plusieurs gros fichiers à l'aide d'une tâche FTP pour chaque téléchargement peut être redémarré après l'échec du téléchargement d'un seul fichier, puis ne reprendre le téléchargement que pour ce fichier.</span><span class="sxs-lookup"><span data-stu-id="9ff81-109">For example, a package that downloads multiple large files by using an FTP task for each download can be restarted after the downloading of a single file fails and then download only that file.</span></span>  
  
-   <span data-ttu-id="9ff81-110">Éviter de répéter le téléchargement de grands volumes de données.</span><span class="sxs-lookup"><span data-stu-id="9ff81-110">Avoid repeating the loading of large amounts of data.</span></span> <span data-ttu-id="9ff81-111">Par exemple, un package qui effectue des insertions en bloc dans des tables de dimension d'un entrepôt de données à l'aide d'une tâche d'insertion en bloc différente pour chaque dimension, peut être redémarré si l'insertion échoue pour une table de dimension, et ne recharger que cette dimension.</span><span class="sxs-lookup"><span data-stu-id="9ff81-111">For example, a package that performs bulk inserts into dimension tables in a data warehouse using a different Bulk Insert task for each dimension can be restarted if the insertion fails for one dimension table, and only that dimension will be reloaded.</span></span>  
  
-   <span data-ttu-id="9ff81-112">Éviter de répéter l'agrégation de valeurs.</span><span class="sxs-lookup"><span data-stu-id="9ff81-112">Avoid repeating the aggregation of values.</span></span> <span data-ttu-id="9ff81-113">Par exemple, un package qui calcule de nombreux agrégats, tels que des moyennes et des sommes, à l'aide d'une tâche de flux de données distincte pour la réalisation de chaque agrégation, peut être redémarré en cas d'échec du calcul d'une agrégation pour ne recalculer que cette agrégation.</span><span class="sxs-lookup"><span data-stu-id="9ff81-113">For example, a package that computes many aggregates, such as averages and sums, using a separate Data Flow task to perform each aggregation, can be restarted after computing an aggregation fails and only that aggregation will be recomputed.</span></span>  
  
 <span data-ttu-id="9ff81-114">Si un package est configuré pour utiliser les points de contrôle, [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] capture le point de redémarrage dans le fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-114">If a package is configured to use checkpoints, [!INCLUDE[ssISnoversion](../../includes/ssisnoversion-md.md)] captures the restart point in the checkpoint file.</span></span> <span data-ttu-id="9ff81-115">Le type de conteneur qui échoue et l'implémentation de fonctionnalités telles que les transactions affectent le point de redémarrage enregistré dans le fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-115">The type of container that fails and the implementation of features such as transactions affect the restart point that is recorded in the checkpoint file.</span></span> <span data-ttu-id="9ff81-116">Les valeurs actuelles des variables sont également capturées dans le fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-116">The current values of variables are also captured in the checkpoint file.</span></span> <span data-ttu-id="9ff81-117">Toutefois, les valeurs des variables de type de données `Object` ne sont pas enregistrées dans les fichiers de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-117">However, the values of variables that have the `Object` data type are not saved in checkpoint files.</span></span>  
  
## <a name="defining-restart-points"></a><span data-ttu-id="9ff81-118">Définition des points de redémarrage</span><span class="sxs-lookup"><span data-stu-id="9ff81-118">Defining Restart Points</span></span>  
 <span data-ttu-id="9ff81-119">Le conteneur d'hôte de tâche, qui encapsule une seule tâche, est la plus petite unité atomique à pouvoir être redémarrée.</span><span class="sxs-lookup"><span data-stu-id="9ff81-119">The task host container, which encapsulates a single task, is the smallest atomic unit of work that can be restarted.</span></span> <span data-ttu-id="9ff81-120">Le conteneur de boucles Foreach et un conteneur transactionnel sont également traités comme des unités atomiques de travail.</span><span class="sxs-lookup"><span data-stu-id="9ff81-120">The Foreach Loop container and a transacted container are also treated as atomic units of work.</span></span>  
  
 <span data-ttu-id="9ff81-121">Si un package est arrêté alors qu'un conteneur transactionnel est en cours d'exécution, la transaction est arrêtée et tout travail effectué par le conteneur est annulé.</span><span class="sxs-lookup"><span data-stu-id="9ff81-121">If a package is stopped while a transacted container is running, the transaction ends and any work performed by the container is rolled back.</span></span> <span data-ttu-id="9ff81-122">Lorsque le package est redémarré, le conteneur qui a échoué est exécuté à nouveau.</span><span class="sxs-lookup"><span data-stu-id="9ff81-122">When the package is restarted, the container that failed is rerun.</span></span> <span data-ttu-id="9ff81-123">La réalisation de conteneurs enfants du conteneur transactionnel n'est pas enregistrée dans le fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-123">The completion of any child containers of transacted container is not recorded in the checkpoint file.</span></span> <span data-ttu-id="9ff81-124">Par conséquent, lorsque le package est redémarré, le conteneur transactionnel et ses conteneurs enfants sont exécutés à nouveau.</span><span class="sxs-lookup"><span data-stu-id="9ff81-124">Therefore, when the package is restarted, the transacted container and its child containers run again.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="9ff81-125">L'utilisation de points de contrôle et de transactions dans le même package peut entraîner des résultats inattendus.</span><span class="sxs-lookup"><span data-stu-id="9ff81-125">Using checkpoints and transactions in the same package could cause unexpected results.</span></span> <span data-ttu-id="9ff81-126">Par exemple, lorsqu'un package échoue et redémarre à partir d'un point de contrôle, il peut répéter une transaction qui a déjà été correctement validée.</span><span class="sxs-lookup"><span data-stu-id="9ff81-126">For example, when a package fails and restarts from a checkpoint, the package might repeat a transaction that has already been successfully committed.</span></span>  
  
 <span data-ttu-id="9ff81-127">Les données de point de contrôle ne sont pas enregistrées pour les conteneurs de boucles For et Foreach.</span><span class="sxs-lookup"><span data-stu-id="9ff81-127">Checkpoint data is not saved for For Loop and Foreach Loop containers.</span></span> <span data-ttu-id="9ff81-128">Lorsqu'un package est redémarré, les conteneurs de boucles For et Foreach et les conteneurs enfants sont exécutés à nouveau.</span><span class="sxs-lookup"><span data-stu-id="9ff81-128">When a package is restarted, the For Loop and Foreach Loop containers and the child containers are run again.</span></span> <span data-ttu-id="9ff81-129">Si un conteneur enfant de la boucle est exécuté correctement, il n'est pas enregistré dans le fichier de point de contrôle et il est exécuté à nouveau.</span><span class="sxs-lookup"><span data-stu-id="9ff81-129">If a child container in the loop runs successfully, it is not recorded in the checkpoint file, instead it is rerun.</span></span> <span data-ttu-id="9ff81-130">Pour plus d'informations et pour obtenir une solution de contournement, consultez [Les points de contrôle SSIS ne sont pas respectés pour les éléments de conteneur de boucles For ou Foreach](https://go.microsoft.com/fwlink/?LinkId=241633).</span><span class="sxs-lookup"><span data-stu-id="9ff81-130">For more information and a workaround, see [SSIS Checkpoints are not honored for For Loop or Foreach Loop container items](https://go.microsoft.com/fwlink/?LinkId=241633).</span></span>  
  
 <span data-ttu-id="9ff81-131">Si le package est redémarré, les configurations du package ne sont pas rechargées, mais le package utilise les informations de configuration écrites dans le fichier de point de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-131">If the package is restarted the package configurations are not reloaded, instead the package uses the configuration information written to the checkpoint file.</span></span> <span data-ttu-id="9ff81-132">Cette procédure assure que le package utilise les mêmes configurations lorsqu'il est exécuté à nouveau qu'au moment où il a échoué.</span><span class="sxs-lookup"><span data-stu-id="9ff81-132">This ensures that the package uses the same configurations when it is rerun as the time it failed.</span></span>  
  
 <span data-ttu-id="9ff81-133">Un package ne peut être redémarré qu'au niveau du flux de contrôle.</span><span class="sxs-lookup"><span data-stu-id="9ff81-133">A package can be restarted only at the control flow level.</span></span> <span data-ttu-id="9ff81-134">Vous ne pouvez pas redémarrer un package au milieu d'un flux de données.</span><span class="sxs-lookup"><span data-stu-id="9ff81-134">You cannot restart a package in the middle of a data flow.</span></span> <span data-ttu-id="9ff81-135">Pour éviter de réexécuter tout le flux de données, le package peut être conçu de manière à inclure plusieurs flux de données, utilisant chacun une tâche de flux de données différente.</span><span class="sxs-lookup"><span data-stu-id="9ff81-135">To avoid rerunning the whole data flow, the package might be designed to include multiple data flows, each one using a different Data Flow task.</span></span> <span data-ttu-id="9ff81-136">De cette manière, le package peut être redémarré en ne réexécutant qu'une seule tâche de flux de données.</span><span class="sxs-lookup"><span data-stu-id="9ff81-136">This way the package can be restarted, rerunning only one Data Flow task.</span></span>
